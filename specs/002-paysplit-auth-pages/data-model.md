# Data Model

## User Entity

The primary entity for the authentication system is the `User`.

**Fields**:
- `_id`: ObjectId (generated by MongoDB)
- `email`: String, Required, Unique, Indexed. Must conform to email regex.
- `name`: String, Required.
- `passwordHash`: String, Optional (Required if signing up via email/password, null if signing up via Google SSO only).
- `googleId`: String, Optional, Unique, Indexed (Required if signing up/linking via Google SSO).
- `createdAt`: Date, auto-generated.
- `updatedAt`: Date, auto-generated.
- `lastLogoutAt`: Date, Optional (used for JWT revocation).

**Validation Rules**:
- `email` must be valid format.
- If `passwordHash` is provided, it must be a bcrypt hash (length/format validation).
- One of `passwordHash` or `googleId` must be present.

**State Transitions**:
- **Creation**: When a user registers via email/password or logs in via Google for the first time.
- **Update**: If an existing email/password user logs in via Google, their `googleId` is updated (account linking).
- **Revocation**: When a user logs out of all sessions, `lastLogoutAt` is set to the current time.
